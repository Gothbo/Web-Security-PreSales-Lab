# 安全产品组合策略：XDR vs SOC/SIP 售前方案匹配指南

> 面向售前工程师的产品组合与客户画像匹配指南——帮助你在不同客户场景下，快速选择正确的产品组合策略。

---

## 一、核心产品认知

### 产品定位速览

| 产品 | 核心能力 | 售前一句话定义 |
|------|---------|--------------|
| **EDR** | 终端检测与响应，主机级威胁定位 | "每台电脑和服务器上的哨兵，能看见黑客在主机上做了什么" |
| **NDR** | 网络检测与响应，流量级威胁分析 | "网络里的摄像头，能看见黑客在内网怎么横向移动" |
| **XDR** | EDR + NDR 原生联动，自动化处置 | "EDR+NDR的合体版，还自带AI帮你自动处理威胁" |
| **SIEM** | 日志统一收集与存储，合规审计 | "全网日志的仓库，满足等保要求的180天日志留存" |
| **SIP/SOC** | 全局态势感知+工单管理+流程编排 | "安全团队的管理后台，资产/漏洞/工单一张大屏看完" |

### 关键认知纠偏

| 常见误解 | 技术实情 |
|---------|---------|
| "SIP/SOC负责告警降噪" | 实际上传统SIEM/SIP是**噪音制造者**，泛收集全网日志导致告警风暴。真正的高精度降噪发生在**XDR层**——XDR通过端网原生数据聚合，把上万条噪点合并成1条有故事线的攻击链，再送给SIP |
| "XDR只是EDR+NDR" | XDR不仅是功能叠加，更是**底层数据原生关联**，实现"秒级自动阻断"这个EDR+NDR各自独立做不到的能力 |
| "大客户才需要SOC" | 任何需要通过等保/监管检查的客户都需要SOC，区别只是建设规模 |

## 二、三大客户画像与方案匹配

### 方案一：轻量实战型 —— XDR

**适用客户画像：**

| 维度 | 特征 |
|------|------|
| **客户类型** | 中小型企业、科技公司、区县级医院、制造业工厂 |
| **安全团队** | 1-3人，严重人手不足 |
| **核心痛点** | 防勒索、防入侵，但没时间天天看告警 |
| **预算范围** | 30-80万 |
| **决策者** | IT主管/安全经理 |
| **合规压力** | 等保二级或刚需等保三级，但主要以实战安全为主 |

**推荐组合：XDR（内置EDR+NDR）**

**售前话术：**
> "老师，咱们安全团队就一两个人，天天看态势感知大屏和几万条报错根本不现实。我们推荐**XDR方案**：它就像一个**全自动的AI巡逻兵+狙击手**。它在底层自动过滤掉99%的网络杂音，一旦发现真正的勒索病毒或黑客入侵，它不需要人工确认，自己就能在毫秒级联动EDR隔离主机、联动防火墙封锁IP。你们只需要在周报里看一眼它自动处置了什么即可，省心省力。"

**亮点总结：**
- ✅ 自动降噪，解放人力
- ✅ 秒级闭环，不等人工
- ✅ 开箱即用，无需专职SOC团队

---

### 方案二：宏观合规型 —— SIP/SOC

**适用客户画像：**

| 维度 | 特征 |
|------|------|
| **客户类型** | 中型国企、三甲医院、市级政务云、上市企业 |
| **安全团队** | 3-8人，有专职安全人员 |
| **核心痛点** | 过等保、应付监管检查、向上级汇报安全工作 |
| **预算范围** | 80-200万 |
| **决策者** | 信息中心主任/CISO |
| **合规压力** | 必须通过等保三级/行业监管检查 |

**推荐组合：SIP/SOC（内置SIEM）**

**售前话术：**
> "领导，针对咱们单位合规审计和上级监管的要求，我们需要搭建一套**SIP/SOC系统**。它能把全网所有服务器、数据库、交换机的日志保存180天以上，满足法律合规红线。同时，它能为您呈现全网的资产暴露面、漏洞整改进度，并生成直观的态势大屏，让安全工作**可量化、可汇报、可审计**。"

**亮点总结：**
- ✅ 180天日志留存，合规检查一次性过
- ✅ 资产/漏洞/工单统一管理，安全可量化
- ✅ 态势大屏，向上汇报有材料

---

### 方案三：顶配御敌型 —— XDR + SIP/SOC

**适用客户画像：**

| 维度 | 特征 |
|------|------|
| **客户类型** | 大型银行、省级电力、头部关基运营商、大型集团 |
| **安全团队** | 8人+，有专属SOC团队 |
| **核心痛点** | 既要合规又要实战，重保/HVV不能失分 |
| **预算范围** | 300万+ |
| **决策者** | CISO/分管领导 |
| **合规压力** | 等保三级及以上、关基保护条例、行业强监管 |

**推荐组合：XDR（战术层）+ SIP/SOC（战略层）**

**售前话术：**
> "各位领导，针对贵行对业务连续性的极致要求和实战重保（如HVV）的需求，我们建议构建**XDR + SIP/SOC的双剑合璧方案**：
>
> * **XDR充当前线特种部队**：部署在端点和网络边界，利用原生关联算法进行极高精度的告警降噪。发现威胁直接自动封锁，实现**秒级止血**，解决传统SOC光看不能动的痛点。
>
> * **SIP/SOC充当后方联合作战司令部**：XDR将清洗后的高价值安全事件推送到SOC，SOC结合贵行的人员组织架构、资产重要程度、业务合规要求进行全局编排（SOAR），下发工单落实后续的系统补丁修复和定点整改。
>
> 这套方案既能保证日常运营的**合规与管理（SOC）**，又能确保在面临真实APT攻击时**打得赢、防得住（XDR）**。"

**亮点总结：**
- ✅ XDR做战术执行——秒级检测+自动阻断
- ✅ SOC做战略管理——合规审计+工单闭环
- ✅ 双剑合璧，攻防演练不失分，监管检查不掉链

## 三、快速决策表

拿到客户信息后，30秒判断推荐哪套方案：

| 判断维度 | 倾向XDR | 倾向SOC | 倾向XDR+SOC |
|---------|---------|---------|------------|
| **安全团队人数** | 1-3人 | 3-8人 | 8人+有SOC团队 |
| **核心需求** | 防勒索/实战 | 合规/汇报 | 两者都要 |
| **预算** | 30-80万 | 80-200万 | 300万+ |
| **合规压力** | 一般 | 强（等保必过） | 极强（关基+HVV） |
| **决策者关注点** | "别出事就行" | "监管要材料" | "既要合规又要安全" |
| **典型行业** | 中小企业/医院/工厂 | 国企/医院/政务 | 银行/电力/头部企业 |

## 四、异议应对话术

| 客户异议 | 背后原因 | 应对话术 |
|---------|---------|---------|
| "我们已经有防火墙了" | 不想重复投资 | "防火墙是边界防护，看不到内网横向移动。XDR覆盖的是防火墙管不到的端点侧和流量侧，是互补不是重复。" |
| "SOC太贵了" | 预算有限 | "那先上XDR，自动降噪+秒级阻断，您1-2个人的团队也能管得过来。等合规需要了再扩建SOC。" |
| "XDR和SOC有什么区别？" | 不了解产品定位 | "XDR是帮你干活的兵，SOC是管兵打仗的司令部。小团队先有兵再考虑司令部，大团队两者都得有。" |
| "我们还在评估阶段" | 观望/比价 | "那我先给您做个轻量POC测试，XDR部署1周就能看到效果。有效果再谈预算。" |

## 六、售前速查：见客户前30秒

```
客户有合规刚需？          → 是 → SOC方案
                         → 否 → XDR方案

客户安全团队是否<3人？    → 是 → XDR（自动处置，解放人力）
                        → 否 → 问下一个问题

客户是否要通过HVV/重保？  → 是 → XDR+SOC双剑合璧
                        → 否 → SOC方案够了

客户预算是否>200万？     → 是 → XDR+SOC全套上
                        → 否 → 根据核心痛点选一个
```

---

> **一句话总结：**
> XDR解决"打不打得赢"的问题，SOC解决"合不合规管不管得住"的问题。
> 小客户先要兵（XDR），大客户既要兵也要司令部（XDR+SOC）。
> 选错方案=客户花冤枉钱，选对方案=客户把你当顾问。
