# 各行业方案敏感度地图（审查升级版）

> 每个行业决策者的"敏感点"完全不同——方案里先写什么、重点强调什么、不敢提什么，都需要行业定制。
> 本文档覆盖：6+1个行业敏感度 × 客户规模分层 × 会中快速识别法 × 季节性敏感度变化。

---

## 一、行业敏感度速查表

| 行业 | 第一敏感 | 第二敏感 | 第三敏感 | 隐藏敏感 | 一句话总结 |
|------|---------|---------|---------|---------|-----------|
| **政府** | 落地时间（什么时候能建好） | 金额（有没有超出预算） | 合规性（能不能过等保/监管检查） | 透明度（每一步必须有据可查） | **"什么时候能建好，多少钱，能不能向上交差"** |
| **金融** | 安全性（数据不会丢） | 合规性（银保监会/人行要求） | 业务影响（会不会导致交易中断） | 审计追溯（每一笔操作都能追） | **"数据安全第一，交易不能断，合规必须过"** |
| **医疗** | 业务连续性（不能停诊） | 设备兼容性（老旧设备能不能用） | 防勒索（这个是最怕的事） | 对现有系统零干扰 | **"设备不能停，数据不能丢，勒索必须防"** |
| **制造** | 生产不中断（产线不能停） | OT/IT隔离方案（怎么不干扰工控网） | 设备兼容性（10年前的PLC能不能管） | 对生产效率的影响 | **"产线第一，IT帮OT，别添乱"** |
| **零售** | 业务不受影响（不能影响收银/销售） | 支付安全（PCI DSS合规） | 客户隐私（C端数据泄露的舆情风险） | 对顾客无感知 | **"不能影响卖货，支付不能出事"** |
| **连锁零售** | 统一管控（100+门店怎么管） | 门店轻量化（每店部署成本和时间） | 分级响应（总部vs门店的响应权限） | 扩展成本（新增门店的边际成本） | **"100家店要像1家店一样管"** |
| **教育** | 预算性价比（经费有限） | 等保合规（高校必须过等保） | 不影响教学（不能影响教务系统） | 维护难度（学校IT团队小） | **"钱少事多，好用不贵，别太复杂"** |

## 二、各行业深度拆解

### 2.1 政府：时间和金额是命门

**方案里最先写什么：**
- **实施周期**：分几期？每期多久？什么时候能看到成果？
- **预算明细**：不是总价，是每一项多少钱（硬件/软件/服务/培训）
- **合规对齐**：对应等保几级？哪个条款？一一对照

**方案里最不能缺的：**
```
实施计划表（月历形式，不要纯文字）：
第1-2月：硬件部署与网络改造
第3-4月：平台搭建与数据对接
第5月：联调测试与优化
第6月：验收与培训交付
```

**方案里不能提的：**
- "长期见效" —— 政府要的是**任期内有成果**，太长没用
- "方案可能调整" —— 政府怕变数，任何不确定性都让他们不敢批
- "根据实际需求调整配置" —— 他们要做的是**固定预算申报**，不确定性没法审批

**有力的话术（不是词汇，是能直接说的句子）：**

| 场景 | 直接说 |
|------|--------|
| 强调交付时间 | "6个月交付，其中前3个月就能看到态势大屏上线" |
| 强调预算可控 | "总预算在XX万以内，分两期执行，首期只需XX万" |
| 强调合规 | "已通过等保三级测评，对应等保2.0第X-X条要求，有测评报告" |
| 强调同行 | "参考案例：XX市大数据局、XX省政务云，规模与贵单位相当" |

**决策链：**
```
签字人：分管副局长（批预算）
影响者：信息中心主任（写需求/选型）
阻碍者：采购中心/审计部门（卡流程）
注意：签字人和影响者的敏感点不同——副局长关心"政治风险"，信息中心主任关心"技术落地"
```

**隐藏敏感——透明度：**
> 政府项目的每一步都要"有据可查"。不是因为你做得好就行，而是**审计来的时候你要能拿出证据**。
> 所以方案里要加"项目里程碑文档交付清单"——每个阶段交付什么文档、谁签字、归档在哪里。
> 这个你不主动提，客户也不会说，但审计的时候会查。

### 2.2 金融：安全和合规是底线

**方案里最先写什么：**
- **数据安全架构**：数据怎么流转、怎么加密、怎么备份
- **合规对照表**：银保监会XX号文、人行XX要求、等保金融扩展要求——全部列出来
- **业务连续性保障**：方案部署对交易系统的影响评估

**方案里最不能缺的：**
```
合规条款对照表（每一条都列出）：
| 监管要求 | 原文条款 | 方案如何满足 |
|---------|---------|-------------|
| 《银行保险机构数据安全办法》第X条 | XXXXX | 数据库加密+审计+脱敏 |
| 《等保2.0金融扩展要求》第X条 | XXXXX | 三级堡垒机+日志审计180天 |
```

**方案里不能提的：**
- "这是我们通用的方案" —— 金融客户要的是**定制化**，通用方案表示你不懂金融
- "部署期间可能需要短暂停机" —— 金融系统不能停，哪怕是凌晨
- "部分数据会经过云端" —— 金融数据不能出境/出域

**有力的话术：**

| 场景 | 直接说 |
|------|--------|
| 强调安全 | "数据全流程加密，密钥由贵行自行管理，我们接触不到明文" |
| 强调合规 | "银保监会9号文要求的6项数据安全能力，我们逐条对标，对照表在第X页" |
| 强调业务连续 | "方案采用旁路部署+热备切换，对交易系统零影响，已有XX银行POC验证" |
| 强调审计 | "所有操作日志留存180天以上，支持一键导出审计报告，满足内外审要求" |

**决策链：**
```
签字人：CISO / 信息科技部总经理
影响者：安全团队负责人（技术评估）、合规部门（合规审查）
阻碍者：业务部门（担心安全方案影响业务效率）
注意：金融行业"安全"和"效率"天然矛盾，业务部门是隐形阻碍者
```

**隐藏敏感——审计追溯：**
> 金融客户嘴上说"要安全"，心里想的是"出事了能不能追到人"。
> 合规审计要求每笔操作可追溯，所以方案里的日志审计和堡垒机不能只写功能，
> 要写"支持按时间/账号/IP/操作类型四维检索，3秒内定位任何一条操作记录"。

### 2.3 医疗：不停诊高于一切

**方案里最先写什么：**
- **业务零影响方案**：部署会不会影响HIS/PACS/EMR系统运行
- **勒索病毒专项防护**：医疗行业头号威胁，必须单独一节
- **老旧设备兼容**：CT机/MRI/监护仪的系统版本和支持情况

**方案里最不能缺的：**
```
设备兼容性清单（不能只写"支持IoMT"，要具体）：
| 设备类型 | 常见系统 | 兼容方案 | 是否需要装Agent |
|---------|---------|---------|----------------|
| CT机 | Windows 7 Embedded | 旁路探针 | 不需要 |
| MRI | Windows XP | 流量镜像 | 不需要 |
| 监护仪 | Linux定制 | SNMP采集 | 不需要 |

医疗行业决策者的核心恐惧：部署了安全方案，结果CT机启动不了了。
```

**方案里不能提的：**
- "需要给每台设备安装Agent" —— 医疗设备不能装，装不了，也不敢装
- "替换现有系统" —— 医疗系统极其复杂，替换是不可能的
- "定期重启" —— 医院设备7×24运行，没有重启窗口

**有力的话术：**

| 场景 | 直接说 |
|------|--------|
| 强调零影响 | "旁路部署，不装软件，不碰设备，对HIS/PACS零干扰" |
| 强调防勒索 | "专门针对WannaCry/新型勒索的横向传播检测，3分钟内自动隔离被感染设备" |
| 强调兼容 | "支持3000+医疗设备型号自动识别，不需要手工录入资产" |
| 强调兜底 | "即使设备被加密，我们有一键恢复到最近备份点的能力，RTO<15分钟" |

**决策链：**
```
签字人：分管副院长（批预算）
影响者：信息科主任（技术选型）
阻碍者：临床科室主任（极度保守，任何可能影响设备的方案都反对）
注意：临床科室主任往往不参加安全方案评审会，但他们的意见能一票否决
```

**隐藏敏感——对现有系统零干扰：**
> 信息科主任嘴上说"要安全"，心里想的是"别给我惹事"。
> 你部署的安全方案如果导致HIS系统卡顿0.5秒，门诊就排长队，
> 临床主任一个电话打到院长那里，信息科主任就得写检讨。
> 所以方案里必须写"对HIS系统响应时间影响<0.01秒，附测试数据"。

### 2.4 制造：产线不能停，这是铁律

**方案里最先写什么：**
- **OT/IT隔离架构**：怎么保护生产网不被IT网的安全事件波及
- **对产线的零影响保障**：方案实施和运行不干扰PLC/DCS/SCADA
- **工控协议支持列表**：你们的探针能解析哪些协议？

**方案里最不能缺的：**
```
OT网络拓扑图（画出隔离边界）：
[IT网] ── 网闸/单向隔离 ── [OT网]
  │                           │
ERP/OA/邮件                PLC/DCS/SCADA
传统探针                   工控探针（旁路）
```

**方案里不能提的：**
- "能给PLC打补丁/升级" —— 工控设备不能随便升级，打了补丁产线停了谁负责
- "全流量采集" —— 产线网络流量不大但极其敏感，采集太多反而有风险
- "AI自动阻断" —— OT环境不接受自动阻断，万一是误判产线就停了

**有力的话术：**

| 场景 | 直接说 |
|------|--------|
| 强调隔离 | "IT/OT物理隔离，安全事件绝不会从办公网扩散到产线" |
| 强调旁路 | "旁路监听，像摄像头一样只看不碰，不改变任何工控网络拓扑" |
| 强调告警语言 | "告警不是报'发现漏洞'，而是报'3号产线PLC温度参数被篡改'——用生产语言说话" |
| 强调不改动 | "不改变任何现有网络架构，不需要给PLC/DCS做任何配置变更" |

**决策链：**
```
签字人：厂长/生产副总
影响者：自动化部主管（技术评估）
阻碍者：车间主任（不接受任何可能影响产线的操作）
注意：车间主任可能不懂安全，但他有一个绝对否决权——"产线不能停"
```

**隐藏敏感——对生产效率的影响：**
> 产线不停 ≠ 不影响效率。有些安全方案虽然不停线，但会引入网络延迟，
> 导致工控指令响应从10ms变成50ms，在高速生产线上这就是次品率上升。
> 方案里要写"对工控网络延迟影响<Xms，附测试数据"。

### 2.5 零售：不能影响卖东西

**方案里最先写什么：**
- **业务连续性**：部署方案不影响POS系统、电商平台、库存系统
- **支付安全**：PCI DSS合规、支付数据加密
- **客户隐私保护**：会员信息、交易记录的防护

**方案里最不能缺的：**
```
业务影响评估表：
| 业务系统 | 部署影响 | 回退方案 | 影响时长 |
|---------|---------|---------|---------|
| POS收银 | 2小时窗口内分批部署 | 切换回旧网络 | <5分钟/店 |
| 电商平台 | WAF接入，无需停机 | DNS切换 | 0影响 |
| 会员系统 | 数据库加密，在线操作 | 主从切换 | <1分钟 |
```

**方案里不能提的：**
- "需要关店部署" —— 零售不可能关店，部署窗口只能在凌晨
- "全网大改造" —— 零售门店多/IT人员少，大规模改造不现实
- "数据上云" —— 客户数据上云在某些零售场景是敏感话题

**有力的话术：**

| 场景 | 直接说 |
|------|--------|
| 强调不影响 | "分阶段部署，先总部再旗舰店再全部门店，每店部署窗口<2小时" |
| 强调支付安全 | "PCI DSS合规方案，支付数据全程加密，满足银联安全认证要求" |
| 强调防泄露 | "会员数据脱敏存储，即使数据库被拖库，拿到的是不可逆的脱敏数据" |
| 强调轻量 | "门店只需部署轻量探针，对收银系统零影响，带宽占用<1Mbps" |

**决策链：**
```
签字人：CIO / IT总监
影响者：门店运营负责人（关心收银不能断）
阻碍者：业务部门（安全=花钱=影响毛利）
注意：零售行业安全预算天然紧张，业务部门永远觉得"安全投入看不见回报"
```

**隐藏敏感——对顾客无感知：**
> 零售的一切都要"顾客无感"。安全方案如果导致：
> - 支付时多了1秒验证 → 顾客嫌慢，下次去别家
> - 会员登录多了验证码 → 顾客放弃登录，流失转化
> - WiFi多了安全页面 → 顾客连不上网，投诉
> 所以方案里要写"安全验证对顾客端零感知，不影响任何用户体验"。

### 2.6 连锁零售：100家店要像1家店一样管

> 连锁零售 ≠ 零售。单店零售的核心是"不影响卖货"，连锁的核心是"如何统一管"。

**方案里最先写什么：**
- **统一管控架构**：总部平台如何同时管理100+门店的安全策略
- **门店轻量化方案**：每家门店部署什么、成本多少、多快能装完
- **分级响应机制**：总部能做什么、门店能做什么、边界在哪里

**方案里最不能缺的：**
```
分级管控架构：
总部平台：统一策略下发 + 全局态势大屏 + 跨店威胁关联分析
  │
  ├── 旗舰店：本地XDR + 独立响应权限
  ├── 标准店：轻量探针 + 策略由总部下发
  └── 小型店：SaaS探针 + 全托管模式

新增门店边际成本：XX万/店（含硬件+部署+首年服务）
```

**方案里不能提的：**
- "每家店独立部署一套完整方案" —— 成本爆炸，100家店就是100套
- "需要每个门店配安全运维人员" —— 不可能，连IT人员都只有1个
- "门店数据全部回传总部" —— 带宽不够，隐私合规也有问题

**有力的话术：**

| 场景 | 直接说 |
|------|--------|
| 强调统一管控 | "总部一张大屏看100家店的安全态势，策略统一下发，门店零运维" |
| 强调边际成本 | "首店部署XX万，后续每新增一家店只需XX万，含硬件和首年服务" |
| 强调分级 | "旗舰店有独立响应权限，标准店策略由总部管控，灵活适配不同规模" |
| 强调快部署 | "单店部署4小时内完成，不影响白天营业，凌晨窗口即可" |

**决策链：**
```
签字人：集团CIO / 信息化总监
影响者：区域IT经理（负责几十家店的运维，最关心省不省心）
阻碍者：门店店长（不接受任何影响门店运营的操作）
注意：区域IT经理是关键影响者——他一个人管几十家店的IT，方案能不能让他少加班，他投谁的票
```

### 2.7 教育：钱少、人少、要求不少

**方案里最先写什么：**
- **性价比**：总投入和产出要清楚，最好有同行对比
- **等保合规**：教育系统/网站需要过等保，这是刚性需求
- **运维简便**：学校IT团队通常只有1-3人，要说明你们能提供多少运维支撑

**方案里最不能缺的：**
```
运维支撑方案（教育行业最怕"买完没人管"）：
| 服务项 | 内容 | 服务方式 |
|-------|------|---------|
| 远程运维 | 7x12小时电话+远程 | 远程接入 |
| 季度巡检 | 现场巡检一次/季度 | 现场 |
| 应急响应 | 4小时内到场 | 现场 |
| 培训 | 每年2次安全培训 | 现场+线上 |
```

**方案里不能提的：**
- "XX万起步" —— 教育预算少，上来报高价直接劝退
- "需要专职安全团队" —— 学校没有，招不到，也养不起
- "复杂的分级权限管理" —— 学校IT管理人员就几个人，不需要大而全的管理体系

**有力的话术：**

| 场景 | 直接说 |
|------|--------|
| 强调性价比 | "同规模高校部署案例，3万人师生规模，总投入XX万，含3年服务" |
| 强调省心 | "托管式服务，7x12小时远程运维+4小时到场应急，学校不需要配专职安全人员" |
| 强调合规 | "等保2.0三级对标方案，已帮助XX所高校通过测评" |
| 强调培训 | "每年2次免费安全培训，含实训环境，帮学校培养安全人才" |

**决策链：**
```
签字人：分管副校长（批预算）
影响者：网络中心主任（技术选型+日常运维）
阻碍者：财务处（卡预算）、教务处（怕影响教学系统）
注意：网络中心主任的权力比想象中大——他既要选型又要运维，
方案能不能让他"少操心"，直接影响他推不推你的方案
```

**隐藏敏感——维护难度：**
> 学校买了方案后，谁来管？网络中心就1-3个人，管着全校网络+系统+安全，
> 根本没有精力再管一套复杂的安全平台。
> 所以方案的核心竞争力不是"功能多"，而是"运维轻"。
> 要写清楚：哪些事情你们帮他做，哪些事情他只需要看结果。

## 三、客户规模分层：同一行业，规模不同，敏感度不同

### 3.1 医疗行业

| 规模 | 代表 | 第一敏感 | 第二敏感 | 方案重点 |
|------|------|---------|---------|---------|
| **三甲医院** | 1000+床位，HIS/PACS/LIS/EMR全有 | 防勒索+设备兼容 | 合规（等保三级必过） | 完整方案+设备兼容性清单+合规对照 |
| **二甲医院** | 500床位，核心系统有但老旧 | 预算+业务不中断 | 合规 | 性价比方案+轻量部署+托管服务 |
| **社区医院** | <100床位，可能只有HIS | 预算 | 能不能用（运维能力） | SaaS化方案+全托管+极简部署 |

### 3.2 制造行业

| 规模 | 代表 | 第一敏感 | 第二敏感 | 方案重点 |
|------|------|---------|---------|---------|
| **头部工厂**（汽车/电子） | 全自动化产线，有自动化部门 | OT/IT隔离+工控协议支持 | 产线零影响 | 完整OT安全方案+工控协议清单 |
| **中型工厂** | 半自动化，IT和OT混合管理 | 产线不中断 | 预算 | IT/OT融合方案+分阶段实施 |
| **小厂** | 手工+少量自动化 | 预算 | 简单能用 | IT网安全为主+OT网基础防护 |

### 3.3 金融行业

| 规模 | 代表 | 第一敏感 | 第二敏感 | 方案重点 |
|------|------|---------|---------|---------|
| **大型银行/交易所** | 交易量百万级EPS | 数据安全+交易零中断 | 合规审计 | 完整方案+SOAR+合规对标 |
| **城商行/农商行** | 交易量万级EPS | 合规（监管检查要过） | 预算 | 合规驱动方案+性价比 |
| **保险公司/证券** | 业务模式不同 | 数据安全 | 合规 | 定制化方案+数据分类分级 |

### 3.4 教育行业

| 规模 | 代表 | 第一敏感 | 第二敏感 | 方案重点 |
|------|------|---------|---------|---------|
| **双一流高校** | 3万+师生 | 合规+科研数据保护 | 统一管控 | 完整方案+等保对标+科研数据保护 |
| **普通本科** | 1-3万师生 | 预算+合规 | 运维简便 | 标准方案+托管服务+培训 |
| **职业院校/中小学** | 规模小，预算极少 | 预算 | 能不能管 | SaaS方案+全托管+极简 |

## 四、会中快速识别法：5分钟判断客户最敏感什么

> 写方案前可以慢慢研究，但见面5分钟内你必须判断客户最关心什么，才能在接下来的55分钟里打对方向。

### 4.1 开场3个问题（替代"您有什么需求"）

```
问题1："X总，咱们单位最近有没有遇到什么安全方面比较头疼的事？"
  → 如果回答是具体事件（被攻击/被通报/被罚款）→ 危机型客户 → 第一敏感=解决眼下问题
  → 如果回答是"还好"或"没什么"→ 常规型客户 → 需要用问题2深挖

问题2："咱们安全建设方面，目前最优先想解决的是哪块？"
  → 如果回答提到"合规/等保/检查"→ 合规驱动 → 重点写合规对照
  → 如果回答提到"勒索/攻击/安全事件"→ 实战驱动 → 重点写威胁检测
  → 如果回答提到"预算/性价比"→ 成本驱动 → 重点写TCO和ROI
  → 如果回答提到"运维/人手不够"→ 运维驱动 → 重点写托管服务和轻量化

问题3："咱们这个项目，是您这边主导，还是需要其他部门一起参与？"
  → 如果回答"我定就行"→ 你在跟决策者说话 → 直接推方案
  → 如果回答"需要领导/XX部门一起看"→ 你在跟影响者说话 → 帮他准备汇报材料
```

### 4.2 观察型判断（客户不说，但你能看到）

| 观察信号 | 说明什么 | 你的应对 |
|---------|---------|---------|
| 客户频繁看手机/表 | 对你不感兴趣或时间紧 | 10分钟内给结论，不要铺垫 |
| 客户反复问"多少钱" | 成本是第一敏感 | 先给价格框架，再讲价值 |
| 客户旁边坐了几个不同部门的人 | 多决策者 | 准备分别打向不同角色的论点 |
| 客户办公室里贴了等保/合规相关文件 | 合规是硬需求 | 方案里加合规对照表 |
| 客户主动提到竞品名称 | 已经在对比了 | 准备差异化对比，不要贬低 |
| 客户问"你们有没有同行业案例" | 他需要安全感/背书 | 准备2-3个同行案例故事 |

## 五、季节性敏感度变化

> 同一个客户，不同时间段敏感度会变。方案要跟着时间调。

| 时间节点 | 受影响行业 | 敏感度变化 | 方案调整 |
|---------|-----------|-----------|---------|
| **HVV/重保期间（每年4-6月）** | 政府、金融、医疗 | 合规和实战敏感度飙升 | 方案增加"HVV专项保障方案"章节，强调应急响应和重保值守能力 |
| **财政年度末（12月）** | 政府、教育 | 预算敏感度下降（花不完的预算要收回） | 推"年底突击采购"方案，强调快速交付 |
| **新财政年度初（1-3月）** | 政府、教育 | 预算敏感度上升（新预算还没批） | 推"分期付款"或"先试用后付费"方案 |
| **等保测评季（通常是Q2/Q4）** | 所有行业 | 合规敏感度飙升 | 方案增加"等保测评辅导+整改一条龙" |
| **勒索病毒高发期（节假日前后）** | 医疗、教育 | 防勒索敏感度飙升 | 推"防勒索专项方案"，强调快速部署 |
| **开学季（9月）** | 教育 | 教务系统不能出任何问题 | 方案强调"零影响部署+教务系统专项保护" |
| **双十一/大促期间** | 零售、连锁零售 | 业务连续性敏感度达到顶点 | 推"大促期间安全保障方案"，7x24值守 |

## 七、方案制作优先级（按行业）

| 行业 | 方案第一章写什么 | 方案核心章节 | 方案最长的章节 |
|------|----------------|------------|--------------|
| **政府** | 项目实施计划与分期方案 | 合规对齐与预算明细 | 实施计划（详细的月度表） |
| **金融** | 数据安全架构与合规对照表 | 业务连续性保障方案 | 合规条款对照（每条都要回应） |
| **医疗** | 业务零影响方案 | 勒索病毒防护与老旧设备兼容 | 设备兼容性清单 |
| **制造** | OT/IT隔离架构 | 产线零影响保障 | 工控协议支持列表 |
| **零售** | 业务连续性保障 | 支付安全与客户隐私 | 分阶段部署方案 |
| **连锁零售** | 统一管控架构 | 门店轻量化与分级响应 | 分级管控+边际成本 |
| **教育** | 性价比分析与同行案例 | 等保合规方案 | 运维支撑与服务方案 |

## 八、一句话记忆口诀

```
政：时间+金额+能交差
金：数据+合规+可追溯
医：不停+防勒+别惹事
造：产线+隔离+别添乱
零：卖货+支付+无感知
连：统管+轻量+边际成本
教：省钱+省心+有人帮
```

> **核心规律：所有行业的敏感点都可以归结为——"这个安全方案，会不会影响我的核心业务"。**
> 政府的核心业务是"向上级交差"，金融是"资金和数据安全"，医疗是"病人能看病"，
> 制造是"产线能转"，零售是"货能卖"，连锁是"100家店统一管"，教育是"课能上"。
> 你的方案如果让他们的核心业务受到任何威胁，不管技术上多牛，都不会被批准。
