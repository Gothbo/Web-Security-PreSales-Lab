## 二、渠道与搜索技巧

**百度搜索指令**（在搜索引擎上高效找案例）：

```
site:sangfor.com.cn "某" "安全" "案例"
site:qianxin.com.cn "解决方案"
site:zhihu.com "深信服" "案例" "售前"
```

**微信公众号**：关注各厂商的官方公众号，定期推送案例文章。

**行业会议演讲**：深信服创新大会、ISC（中国互联网安全大会）、CSS（中国网络安全大会）的演讲PPT中通常包含详细案例。

## 三、如何判断一个案例值不值得学

| 好案例的特征 | 差的案例的特征 |
|-------------|---------------|
| 有具体的客户行业和规模 | "某客户""某企业"过于模糊 |
| 痛点描述清晰有画面感 | 只有技术术语堆砌 |
| 有量化成果（时间/比例/数量） | "效果显著""大幅提升"没数字 |
| 讲清楚了"为什么选我" | 只列产品清单不解释逻辑 |
| 可迁移到类似客户场景 | 场景太特殊，无法复用 |

## 四、内化学习计划

```
第1周：搜10个案例，用模板填3个
第2周：选2个你最熟悉的，写成面试话术反复练
第3周：尝试不看模板，能完整讲出3个案例
第4周：试着把案例应用到新场景中
       （如"美宜佳的MSS逻辑能不能套到连锁药店？"）
```

## 五、面试中如何呈现案例

```
面试官问："你有没有什么项目经验？"

❌ 差的回答：
"我之前做过一个XX的项目，用了防火墙、WAF、态势感知..."
（听的人记不住，没有画面感）

✅ 好的回答：
"我印象最深的是美宜佳便利店的案例。他们数万家门店，
安全就三个人兼着，节假日最怕出事。我们的MSS方案帮他们
把95%的安全运营工作接管了，响应时间从数小时降到7分钟。
这个案例让我明白了，客户要的不是一堆设备，而是省心。"
（有背景、有痛点、有数字、有感悟）
```
