# 制造业工控安全售前产品指南

> 面向初级售前工程师的制造业/工控客户售前指南——从OT安全特点、典型攻击链、产品方案到提案话术的完整链路。

---

## 一、制造业客户画像

### 核心特征

| 维度 | 制造业特点 | 对售前的影响 |
|------|-----------|-------------|
| **IT/OT融合** | 传统IT网络（办公）+ OT网络（工控）长期割裂，正在推进融合 | 方案必须**分IT和OT两张网**，不能一套方案打天下 |
| **高实时性要求** | PLC/DCS对毫秒级延迟敏感，不允许网络抖动 | **任何安全方案不能影响生产节拍**，"旁路部署""不影响OT流量"是关键 |
| **设备老旧** | PLC/SCADA可能运行10-20年，操作系统是Windows XP甚至更老 | 很多安全软件**装不了**，必须用网络层/物理层方案 |
| **停机成本极高** | 停产1小时损失数十万至数百万（汽车零部件停产按分钟计） | "业务连续性"是**压倒一切**的考量，安全是第二位的 |
| **人才短缺** | 懂安全的人不懂PLC，懂PLC的人不懂安全 | 售前要同时懂**IT安全和OT协议**，方案要能"翻译成厂长能懂的话" |

### 制造业IT/OT架构全景

```
                ┌─────────────┐
                │  ERP/PLM/MES│  ← IT层（传统IT安全覆盖）
                │  办公网络   │
                └──────┬──────┘
                       │ 防火墙/网闸隔离
              ┌────────▼────────┐
              │   工业DMZ区     │  ← IT/OT边界（安全重点）
              │  历史服务器     │
              │  工程师站       │
              └────────┬────────
                       │ 单向隔离/协议过滤
        ┌──────────────┼──────────────┐
        │              │              │
   ┌────▼────┐    ┌────▼────┐    ┌────▼────┐
   │ SCADA   │    │ DCS     │    │ PLC/RTU │  ← OT层（工控安全盲区）
   │ 监控层  │    │ 控制层  │    │ 设备层   │
   │ HMI     │    │ I/O模块 │    │ 传感器   │
   └─────────┘    └─────────┘    └─────────┘
        ▲              ▲              ▲
        └──────────────┼──────────────┘
                       │
              ┌────────▼────────┐
              │  工业协议        │
              │  Modbus/OPC     │
              │  PROFINET       │
              │  EtherNet/IP    │
              └─────────────────┘
```

## 二、制造业安全事件应急救火方案

当制造客户**正在发生**安全事件时，售前必须立刻切入：

### 场景A：PLC被篡改导致产品报废

| 维度 | 内容 |
|------|------|
| **客户痛点** | PLC程序被恶意修改，导致2000件产品报废，直接损失数百万 |
| **应急动作** | 隔离被篡改PLC → 恢复原始程序 → 全线排查类似设备 |
| **产品方案** | **工控防火墙 + PLC白名单 + 工控协议审计** |
| **部署方式** | 在PLC与工程师站之间部署工控防火墙，只允许合法协议通信 |
| **报价参考** | 工控防火墙单台约10-30万，按产线数量配置 |
| **售前话术** | "XX汽车零部件厂PLC被篡改导致2000件产品报废。我们用工控防火墙把PLC锁死，只允许授权IP和协议通过，上线后再没出过事。" |

### 场景B：勒索病毒通过IT网络蔓延到OT

| 维度 | 内容 |
|------|------|
| **客户痛点** | IT网络感染勒索病毒，蔓延到SCADA服务器，生产线被迫停摆 |
| **应急动作** | IT/OT物理断网 → IT侧杀毒 → OT侧恢复备份 → 逐步恢复连接 |
| **产品方案** | **工业网闸 + OT网络微隔离 + 工控主机加固** |
| **部署方式** | IT/OT边界部署网闸实现单向隔离，OT内部部署微隔离限制横向移动 |
| **报价参考** | 工业网闸 20-50万，微隔离系统 50-100万（按节点） |
| **售前话术** | "XX化工集团因IT网络勒索病毒蔓延到DCS系统导致停产3天。我们在IT/OT边界部署了工业网闸，OT内部做了微隔离，病毒再也过不去。" |

### 场景C：外部黑客通过互联网入口入侵SCADA

| 维度 | 内容 |
|------|------|
| **客户痛点** | 远程运维VPN被攻破，黑客直接进入SCADA系统控制阀门/温度 |
| **应急动作** | 关闭外部访问通道 → 排查入侵路径 → 强化远程访问管控 |
| **产品方案** | **工业安全接入网关 + 双因素认证 + 操作审计** |
| **部署方式** | 所有外部访问必须经过安全网关，强制MFA，所有操作录屏审计 |
| **报价参考** | 安全网关 + 堡垒机 + 审计系统：80-150万 |
| **售前话术** | "XX水厂被黑客通过远程运维入口入侵SCADA，差点造成水质安全事故。我们用工业安全网关把远程通道全部管起来，黑客进不来了。" |

## 三、常态化产品方案（四大核心方案）

### 方案1：IT/OT边界防护

**解决的问题：** IT网络威胁蔓延到OT生产网络

| 产品组件 | 功能 | 制造业部署要点 |
|---------|------|---------------|
| **工业网闸** | IT/OT之间单向数据隔离，物理层断开 | 替代传统防火墙，实现"数据单向流动" |
| **工控防火墙** | 基于工业协议深度解析，阻断非法指令 | 支持Modbus/OPC/PROFINET等协议识别 |
| **DMZ区隔离** | 在IT和OT之间建立缓冲区域 | 历史服务器/工程师站放在DMZ，核心PLC不直连IT |

> **售前话术**：
> "IT和OT之间必须有物理隔离。XX汽车零部件厂被黑就是因为IT网络感染后直接蔓延到PLC。我们的工业网闸实现单向数据流动——IT可以看OT的数据，但OT不被IT影响。"
>
> **部署原则**：先做"透明桥接"观察模式，确认不影响OT流量后切换为隔离模式。

### 方案2：OT网络内部微隔离

**解决的问题：** 一旦一台PLC被攻破，黑客可以横向移动到整个车间

| 产品组件 | 功能 | 制造业部署要点 |
|---------|------|---------------|
| **OT微隔离系统** | 车间内部PLC/SCADA/HMI之间细粒度隔离 | 基于网络虚拟化，无需在设备上装Agent |
| **工业交换机安全** | 端口安全、MAC绑定、VLAN隔离 | 在工业交换机层面做基础隔离 |
| **白名单机制** | 只允许已知设备通信，未知设备直接阻断 | 先白名单学习期，再切换为强制模式 |

> **售前话术**：
> "XX汽车厂的PLC之间没有任何隔离，一台被黑整条产线瘫痪。我们的微隔离方案不需要在PLC上装任何东西，通过网络层就把通信边界划清楚了。就算一台PLC被黑，黑客也过不去第二台。"

### 方案3：工控主机安全加固

**解决的问题：** SCADA服务器/工程师站操作系统老旧，补丁打不了，漏洞一大堆

| 产品组件 | 功能 | 制造业部署要点 |
|---------|------|---------------|
| **工控主机卫士** | 白名单模式运行，禁止未授权程序 | 适合Windows XP/Win7等老旧系统 |
| **USB管控** | 管控工程师站的U盘/移动硬盘 | 防止通过U盘传播病毒 |
| **外设管控** | 管控串口/并口等工业外设 | 防止通过串口注入恶意指令 |

> **售前话术**：
> "XX化工集团的SCADA服务器跑的是Windows XP，微软早就不管了，补丁打不了。我们的工控主机卫士用白名单模式——只允许已知程序运行，其他一律拦截。等于给老系统套了个安全壳。"

### 方案4：工控安全态势感知

**解决的问题：** 厂长/安全经理不知道OT网络里发生了什么

| 产品组件 | 功能 | 制造业部署要点 |
|---------|------|---------------|
| **工控流量探针** | 旁路镜像OT流量，不干扰生产 | 支持工业协议深度解析 |
| **威胁检测引擎** | 基于工控协议异常检测攻击 | 识别Modbus非法写指令、OPC异常访问 |
| **统一管控平台** | IT安全 + OT安全统一可视 | 厂长看一张大屏就够了 |

> **售前话术**：
> "OT网络以前是黑盒子，出了问题才知道。我们的态势感知平台把OT流量全部镜像分析，PLC被改了程序、SCADA有异常访问，大屏上实时告警。XX汽车厂上线后，安全团队终于知道OT里发生了什么。"

## 四、按制造子行业细化方案

不同制造业的安全需求差异很大，售前必须"分行业讲"：

### 汽车制造

| 关注点 | 安全方案 |
|-------|---------|
| 机器人焊接/装配PLC保护 | 工控防火墙 + 微隔离 |
| MES系统防篡改 | 数据库审计 + 应用安全 |
| 供应链系统安全 | API安全 + 第三方接入管控 |

### 化工/石化

| 关注点 | 安全方案 |
|-------|---------|
| DCS系统防入侵 | 工业网闸 + 工控主机加固 |
| 安全仪表系统(SIS)保护 | 物理隔离 + 白名单 |
| 远程运维安全 | 安全接入网关 + MFA + 操作审计 |

### 电子制造

| 关注点 | 安全方案 |
|-------|---------|
| 核心工艺配方保护 | 数据防泄露(DLP) + 加密存储 |
| 检测设备联网安全 | 工业协议防火墙 |
| 客户数据防泄露 | 终端DLP + 邮件审计 |

### 食品饮料

| 关注点 | 安全方案 |
|-------|---------|
| 生产配方防泄露 | 数据分类分级 + DLP |
| 批次追溯系统安全 | 数据库安全 + 备份 |
| 监管合规(食品追溯) | 日志审计 + 等保整改 |

## 五、合规与标准要求

制造业虽不像金融那样强监管，但也有自己的合规要求：

### 等保2.0 工业控制系统扩展要求

| 等保要求 | 对应产品 |
|---------|---------|
| 网络架构安全 | 工业网闸 + 防火墙 + VLAN隔离 |
| 通信传输安全 | 工业协议加密 + 安全网关 |
| 区域边界安全 | 工控防火墙 + 微隔离 |
| 计算环境安全 | 工控主机卫士 + 白名单 |
| 安全管理中心 | 态势感知平台 |

### 行业标准

| 标准 | 说明 | 对应方案 |
|------|------|---------|
| **IEC 62443** | 国际工控安全标准 | 按等级要求部署IT/OT隔离、访问控制、监控审计 |
| **GB/T 30976** | 中国工控安全标准 | 等保2.0工控扩展要求的国标落地 |
| **NIST SP 800-82** | 美国工控安全指南 | 参考其分区/隔离/监控建议 |

> **售前话术**：
> "工信部要求关键信息基础设施运营者必须落实工控安全防护。等保2.0工控扩展要求明确规定IT/OT必须隔离。我们的方案完全对标等保和IEC 62443要求，测评一次过。"

## 六、制造业售前提案方法论

### 提案逻辑（四步走）

```
1. 痛点切入
   → "XX汽车零部件厂PLC被篡改，2000件产品报废，直接损失300万"
   → "XX化工集团勒索病毒蔓延到DCS，停产3天损失千万"

2. 架构分析
   → "您IT和OT之间现在是什么隔离方式？PLC之间有没有做过隔离？"
   → 用四维追问框架（环境/时间/资产/预算）摸清家底

3. 方案设计
   → "我们的方案分三层：边界防护（网闸）→ 内部隔离（微隔离）→ 态势感知（统一可视）"
   → "全部旁路部署，不影响生产节拍"

4. 投资回报
   → "停产1小时损失XX万，安全投入相当于停产X小时的损失"
   → "XX汽车厂上线后安全事件为零，通过等保测评"
```

### 报价策略参考

| 方案类型 | 投资规模 | 适用企业 |
|---------|---------|---------|
| 基础版（边界防护） | 30-80 万 | 中小企业，IT/OT刚融合 |
| 标准版（边界+微隔离） | 100-200 万 | 中大型企业，有多条产线 |
| 全面版（全栈+态势感知） | 300万+ | 集团化企业，多工厂 |

### 两个版本材料

| 版本 | 受众 | 内容重点 |
|------|------|---------|
| **技术版** | 安全团队、IT部门 | 工业协议解析、部署拓扑、兼容性 |
| **厂长版** | 厂长/总经理 | 停产损失对比、合规要求、同行案例 |

## 七、常见异议与应对

| 客户异议 | 背后原因 | 应对话术 |
|---------|---------|---------|
| "我们产线不能停，装不了" | 怕影响生产 | "我们的方案全部旁路部署，不经过生产流量。XX汽车厂部署时产线零中断。" |
| "PLC太老了，装不了安全软件" | 设备老旧 | "正因为装不了才需要网络层方案。工控防火墙在交换机层面做隔离，不用碰PLC。" |
| "IT安全厂商不懂OT" | 对IT厂商不信任 | "我们有专门的工控安全团队，懂Modbus/PROFINET/OPC等工业协议。方案里每个产品都经过OT环境验证。" |
| "现在没出事，不急" | 缺乏紧迫感 | "等出事就晚了。XX厂被黑前也觉得不急，结果PLC被篡改损失300万。等保2.0也要求工控系统必须做安全防护。" |
| "太贵了" | 预算不足 | "先做IT/OT边界隔离，这是最关键的一步。投入约30万，先把大门守住。微隔离和态势感知可以二期再做。" |

## 八、制造业售前速查清单

面制造客户前，按这个清单准备：

- [ ] 了解该制造企业的子行业（汽车/化工/电子/食品？）
- [ ] 了解产线数量、PLC/SCADA品牌（西门子/AB/三菱？）
- [ ] 了解IT/OT融合程度（完全隔离/部分融合/已融合？）
- [ ] 查近期该行业是否有安全事件（新闻/通报）
- [ ] 准备同行业案例（模糊化处理）
- [ ] 准备等保2.0工控扩展要求对标材料
- [ ] 准备两个版本PPT（技术版 + 厂长版）
- [ ] 了解客户现有工业交换机品牌（利旧方案需要）
- [ ] 确认决策链（IT部门→安全部门→厂长/总经理）

## 九、制造业售前术语表

初级售前必须掌握这些术语，才能在客户面前显得专业：

| 术语 | 含义 |
|------|------|
| **PLC** | 可编程逻辑控制器，产线的"大脑" |
| **DCS** | 分布式控制系统，化工行业常用 |
| **SCADA** | 数据采集与监控系统，OT层监控平台 |
| **HMI** | 人机界面，操作员看数据的屏幕 |
| **MES** | 制造执行系统，IT层的生产管理系统 |
| **工业协议** | Modbus、OPC、PROFINET、EtherNet/IP等OT层通信协议 |
| **IT/OT融合** | 传统IT网络和OT工控网络打通 |
| **工业网闸** | IT/OT之间单向数据隔离设备 |
| **工控防火墙** | 基于工业协议深度解析的防火墙 |
| **等保2.0工控扩展** | 等保2.0针对工业控制系统的补充要求 |

---

> **一句话总结：**
> 制造业售前不是卖产品，是帮客户在**生产连续性**和**安全风险**之间找到平衡点。
> 方案再好，如果厂长觉得"可能让产线停下来"，一切归零。
