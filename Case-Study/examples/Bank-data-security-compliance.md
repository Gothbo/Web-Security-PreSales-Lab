# 售前案例学习笔记：某城商行数据安全合规整改
> *本案例基于公开安全事件与行业通行做法，结合我个人对售前的理解整理而成。*
> *文末附有我的复盘思考。*

---

## 一、基础信息

| 字段 | 内容 |
|------|------|
| 客户名称 | 某城市商业银行（脱敏） |
| 所属行业 | 金融（银行） |
| 客户规模 | 总行+120家分支行，2000+台服务器 |
| 案例来源 | 售前模拟演练场景 |
| 案例日期 | 2026年 |
| 厂商名称 | 深信服 |

---

## 二、客户背景

- **现状描述**：银保监会下发整改通知，要求三个月内完成数据安全合规整改。当前数据分类分级不清，敏感数据在测试环境大量使用真实数据，存在泄露风险。总行安全团队8人，分支机构无专职安全人员。
- **已有安全投入**：已部署防火墙、堡垒机、数据库审计，但数据安全防护体系不完整。
- **行业监管要求**：《商业银行信息科技风险管理指引》《金融数据安全 数据安全分级指南》《个人信息保护法》，等保三级要求。

---

## 三、我的思考

- **打动客户的点**：2周出合规差距报告——客户最焦虑的是"监管要材料"，先给报告等于给了定心丸
- **简化表达**：合规驱动 + 总分行一体化 + 利旧+新增，三句话讲清楚方案价值
- **可迁移场景**：保险、证券、信托等所有受监管的金融机构
- **竞品攻击点**："你们DLP和其他厂商有什么区别？" 防守："我们方案的核心不是DLP一个产品，而是总分行一体化管控能力，这是竞品做不到的"
- **学到的技巧**：金融行业必须懂监管条款——方案里每一项都要对应具体法规，否则客户不买账

---

## 四、痛点分析

**一句话痛点：** 监管要求三个月整改到位，但数据分类分级不清、测试环境用真实数据、分支机构没人管安全。

| 表面诉求 | 深层痛点 | 翻译为技术需求 |
|----------|---------|--------------|
| "监管下发整改通知，要求三个月内完成" | 合规压力巨大，不整改可能被处罚，但内部缺乏专业能力 | 需要快速出合规差距分析报告，给监管和领导一个交代 |
| "数据分类分级不清" | 不知道哪些是敏感数据、存在哪里、谁在用 | 需要数据资产盘点 + 分类分级工具 + 数据流向图 |
| "测试环境大量使用真实客户数据" | 开发流程不规范，缺乏数据脱敏机制 | 需要测试数据脱敏方案 + 数据使用审批流程 |
| "总行8人管安全，分支机构120家没人管" | 安全能力无法下沉，分支行安全管理真空 | 需要总行统一管控平台 + 分支机构自动化巡检 |

---

## 五、解决方案

| 产品/服务 | 解决什么问题 | 为什么选这个 |
|-----------|------------|-------------|
| 数据资产盘点服务（2周） | 梳理全行数据资产，完成分类分级 | 合规第一步，摸清家底 |
| DLP数据防泄露系统（总行+分支） | 敏感数据外发拦截，测试数据脱敏 | 解决真实数据滥用问题 |
| 数据库审计增强（现有设备策略调优） | 敏感数据访问监控和告警 | 利用已有投资，快速见效 |
| 总行统一安全管理平台 | 120家分支行安全状况统一可视 | 解决"分支机构没人管"的痛点 |
| MSS安全运营托管 | 7×24安全监控，弥补8人团队不足 | 释放人力聚焦合规整改 |

**方案亮点：**

- **先摸底再整改**——用2周做数据资产盘点，出合规差距报告，给客户向监管交代的材料
- **总分行一体化**——总行统一管控 + 分支机构自动巡检，解决"120家分支行没人管"的困境
- **利旧+新增结合**——数据库审计用现有设备调优，不重复投资
- **合规驱动**——方案直接对标监管要求，每一项都有对应的合规条款

## 六、量化成果

| 指标 | 之前 | 之后 | 提升效果 |
|------|------|------|---------|
| 数据资产可见性 | 不知道敏感数据在哪 | 全行数据资产清单 + 分类分级标签 | 从"两眼一抹黑"到"一清二楚" |
| 测试数据合规率 | 大量使用真实数据 | 100%测试数据脱敏 | 消除合规风险 |
| 分支行安全覆盖 | 120家分支行无人管 | 总行统一平台覆盖100%分支行 | 安全管理无死角 |
| 安全团队负担 | 8人疲于应付 | MSS托管 + 自动化巡检释放60%人力 | 聚焦核心合规工作 |
| 监管合规达标 | 差距大，整改通知 | 三个月内完成整改，通过监管验收 | 从"不合规"到"达标" |

**客户反馈：**

> "以前是监管查什么我们补什么，现在是主动知道风险在哪。总分行一体化平台上线后，分支行的安全状况一目了然。"
>
> ——某城商行信息科技部总经理

```markdown
"我之前接触过金融行业的一个项目，客户是一家有120家分支行的城商行，
总行安全团队8个人。他们遇到的核心问题是：
银保监会下发整改通知，要求三个月内完成数据安全合规整改，

'以前是监管查什么我们补什么，现在是主动知道风险在哪。'

这个案例让我对金融行业的安全需求有了深刻理解，总结下来就是：
金融行业安全的核心逻辑是'合规先行'——方案必须对标监管要求，
每一项都要有对应的法规依据。
"
```
